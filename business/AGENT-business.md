# AGENT-business

## Purpose

Define reusable venture-building and business-operations workflows.

## Workflow

1. Start from explicit problem and customer segment.
2. Break execution into short, testable phases.
3. Record assumptions, experiments, and outcomes.
4. Convert validated work into repeatable offers/processes.
5. Track next concrete action for each active venture.

## Execution Style

- Prefer small validation loops over large speculative plans.
- Include first actions with realistic time estimates.
- Keep deliverables scoped for solo founder/small-team execution.
- Use sprint-friendly decomposition for ambiguous work.

## Documentation Contract

- Keep venture plans, status, and milestones up to date.
- Prefer measurable outcomes over narrative-only progress logs.
- Keep a clear link from strategy to weekly execution actions.
- Update top-level venture index docs when new ventures are added.

## Guardrails

- Avoid vague “build everything” plans.
- Bias toward small validation loops before scaling.
- Keep public docs generic; private PII/company details stay in
  `<private-repo>`.
