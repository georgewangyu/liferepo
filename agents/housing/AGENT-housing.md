# AGENT-housing

## Purpose

Define reusable workflows for organizing housing/property records and analyses.

## Organization Rules

- Keep property documents grouped by `location -> property -> document type`.
- Keep rental, ownership, and mortgage contexts clearly separated.
- Use consistent naming: date-first where possible and stable property labels.
- Keep records, media assets, and analysis outputs in separate subfolders.

Use `PROPERTY_RECORDS_LAYOUT.md` for canonical folder design.

## Analysis Rules

- Distinguish facts (documents, terms, dates) from interpretation.
- Track assumptions and unknowns explicitly in analysis outputs.
- Use structured checklists for renewals, disputes, and decisions.
- For investment analysis, use `REAL_ESTATE_ANALYSIS_WORKFLOW.md`.

## Privacy

All property files, legal docs, and identity-linked records remain in
`<private-repo>`.
