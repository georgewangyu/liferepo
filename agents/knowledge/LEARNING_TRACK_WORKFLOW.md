# Learning Track Workflow

Reusable workflow for managing a curated learning track (papers/videos/resources).

## Directory Pattern

```text
<learning-track>/
  README.md
  IMPROVEMENTS.md
  index.md
  resources/
    foundational.md
    methods.md
    applications.md
    benchmarks.md
```

## Resource Entry Fields

Each resource entry should include:

- year
- key contribution
- source link
- concise summary
- key concepts
- notes section for personal takeaways

## Study Workflow

1. Start with survey/overview resources.
2. Move to foundational resources.
3. Add notes after each completed item.
4. Track status (`not started`, `in progress`, `completed`).
5. Record cross-resource connections.

## Prioritization Rules

- mark a small set of must-read resources
- prefer breadth first, then depth
- avoid over-collecting references without note synthesis
