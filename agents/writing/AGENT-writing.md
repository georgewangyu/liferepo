# AGENT-writing

## Purpose

Define reusable writing-system behavior for short-form, long-form, and
professional writing tasks.

## Workflow

1. Identify audience, intent, and output channel.
2. Choose structure before writing prose.
3. Draft first pass with one clear argument.
4. Run revision pass for clarity, brevity, and tone.
5. Run final quality pass (facts, formatting, CTA).

## Long-Form Standard

- Use staged flow:
  - planning
  - drafting
  - review
  - publish preparation
- Carry one central argument throughout.
- Balance narrative, evidence, and practical takeaway.
- Use `LONG_FORM_WORKFLOW.md` for phase-level execution.

## Prompt Frameworks

Use these reusable prompt frameworks when drafting with AI:

- `4C` planning: context, command, constraints, creativity.
- stepwise decomposition for complex sections.
- section-level prompts that require point, evidence, transition, and reader value.

## Quality Rules

- Avoid filler and vague claims.
- Prefer concrete examples and measurable statements.
- Keep formatting and readability consistent.
- Match tone to channel while preserving authenticity.
- Keep claims defensible and source-backed when factual.

## Repository Boundary

- Public writing specs live in `liferepo/agents/writing/`.
- User-specific voice profiles and private writing archives stay in
  `<private-repo>/writing/`.
